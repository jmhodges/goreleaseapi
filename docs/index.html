Go release JSON API

The Go release JSON API provides programmatic access to the release versions, and artifacts of the Go programming language.

The API's URLs

/api/latest_version.txt: The latest version of Go in text. It's format matches the third column of the output of go version.

/api/latest_rlease.json: The Release object for the latest version of Go.

/api/all_versions.txt: All of the Go versions sorted in decreasing release version and separated by newlines. The version strings format matches the third column of the output of go version.

/api/all_versions.json: All of the Go versions in JSON format with a top-level versions key pointing to an array of JSON objects with version and link keys. The value of link is a relative HTTP path to aJSON Release objects.

/api/versions/$GO_VERSION/release.json: The JSON Release object for the given $GO_VERSION. Specific Release will be linked to from /api/all_versions.json so you don't have to get the version formatting correct.

The API's objects

Release objects have a artifacts key whose value is an array of Artifact objects and whose version key is the Go version of the release.

Artifact objects have these properties:

version
The short version string of the Go release the artifact is from.
link
The URL where the artifact can be downloaded.
kind
The kind can be either Source (it's the source code for the artifact), Installer (a binary installer tool for a given platform), or Archive (a compiled form of the Go release for that platform)
os
The operating system the artifact is meant to be used on. (if a specific one exists).
arch
The CPU architecture is meant to be used on (if a specific one exists).
size
The number of bytes the artifact is in a human-readable form.
sha256
The SHA256 of the artifact to be used for verification of a correct download.
